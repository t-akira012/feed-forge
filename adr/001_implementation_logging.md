# ADR-001: 実装タスクと結果の記録方式

## ステータス
承認済み (2026-03-13)

## コンテキスト
Claude Codeによる実装作業の経緯・判断・結果を後から追跡できる必要がある。会話コンテキストは揮発するため、ファイルとして永続化する仕組みが必要。

## 決定
`make_log/` ディレクトリに、実装単位ごとにタスクファイルとログファイルを記録する。

### ファイル命名規則
```
make_log/
├── {NNN}_{name}_task.md   # タスクリスト（実装前に作成、完了時にチェック）
└── {NNN}_{name}_log.md    # 実装ログ（実装完了後に記録）
```

- `{NNN}`: 3桁連番（001, 002, ...）
- `{name}`: 実装対象の識別名（例: hatebu, slack_deliver）
- task と log は同じ番号で対にする

### task ファイルの内容
- 対象URL・使用コネクタなどの前提
- チェックボックス付きタスク一覧
- 補足・判断の前提

### log ファイルの内容
- 実装の流れ（時系列）
- TDDでのつまずきポイントと解決策
- 成果物一覧
- 依存パッケージ
- 判断メモ（なぜその方式を選んだか）

## 結果
- 実装の再現性・追跡性が確保される
- 次回の実装時に過去の判断を参照できる
- 会話コンテキストが消えても記録が残る
