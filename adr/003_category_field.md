# ADR-003: input_list.txt の category フィールドによるカテゴリ別出力

## ステータス
承認済み (2026-03-13)

## コンテキスト
従来のGenerateステップではClaude がRSS取得結果から記事内容を読み取り、独自にカテゴリ分類していた。
しかしユーザーが意図するカテゴリ体系（IT, 投資, Game等）とClaude の推測が一致しない場合がある。

## 決定
`input_list.txt` に `category` フィールドを追加し、ユーザーが明示的にカテゴリを宣言する。

### input_list.txt フォーマット
```
---
list: <URL>
conecter_type: <スキル名>
category: <カテゴリ名>
lang: <言語コード（省略可）>
get_data: "<取得指示>"
block_data: "<除外指示>"
---
```

### 出力グループ化ルール
- Generateステップで `category` フィールドの値ごとにグループ化
- `## カテゴリ名` はユーザー宣言値をそのまま使用
- 同一カテゴリに複数ソース（複数URL）の記事が混在してよい
- ソース（ウェブサイト）単位の分類は行わない

## 結果
- ユーザーが出力のカテゴリ体系を完全に制御できる
- 複数ソースを同一カテゴリに集約可能（例: はてブIT + Hacker News → IT）
- Claude の推測に依存しない確定的なグループ化
